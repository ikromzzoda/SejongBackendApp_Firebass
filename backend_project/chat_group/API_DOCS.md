# Chat API Documentation

Групповой чат: студенты одной группы пишут друг другу. Сообщения хранятся в
Firestore (`groups/{group_id}/chat_messages`), отправка идёт через Django,
а **получение в реальном времени** — напрямую из Firestore через
snapshot-листенер (без пушей и без поллинга).

Base URL: `/api/chat/`

Все эндпоинты требуют заголовок:
```
Authorization: Bearer <JWT-токен>
```

Группа пользователя определяется по полю `group` его документа в Firestore
(актуальное членство, не из JWT). Если пользователь не состоит в группе —
`403 {"error": "Вы не состоите ни в одной группе"}`.

---

## Как клиент получает сообщения мгновенно (realtime)

1. После логина в приложение вызвать `GET /api/chat/firebase-token/`.
2. Войти в Firebase: `signInWithCustomToken(firebase_token)`.
3. Повесить листенер на сабколлекцию своей группы:

```dart
// Flutter (cloud_firestore)
FirebaseFirestore.instance
    .collection('groups')
    .doc(groupId)
    .collection('chat_messages')
    .orderBy('created_at', descending: true)
    .limit(50)
    .snapshots()
    .listen((snap) { /* обновить UI */ });
```

Новые сообщения прилетают в листенер сами — GET-запросы не нужны.
Сессия Firebase продлевается SDK автоматически; custom token нужен только
для входа. Запрашивайте свежий токен при каждом запуске приложения (или при
ошибке доступа) — так подхватывается перевод студента в другую группу.

**Отправка** — всегда через `POST /api/chat/messages/` (Django проверяет
членство и валидирует текст). Прямая запись в Firestore запрещена правилами.

## Прочитанность («кто посмотрел», как в Telegram)

Хранится не список просмотров на каждое сообщение, а **указатель чтения**
на пользователя: `groups/{group_id}/chat_read_status/{user_id}` с полем
`last_read_at`. Сообщение просмотрено пользователем, если
`message.created_at <= last_read_at`.

Обязанности клиента:

1. Когда пользователь реально видит сообщения (чат открыт, приложение на
   переднем плане, новые сообщения отрисованы) — вызвать
   `POST /api/chat/read/`. Практично: при открытии чата и при каждом новом
   сообщении, пришедшем в открытый чат (можно троттлить до раза в пару секунд).
2. Для галочек «просмотрено» повесить второй листенер на
   `groups/{group_id}/chat_read_status` — обновления указателей приходят в
   реальном времени, и клиент сам вычисляет, кто уже видел каждое сообщение.
   REST-фолбэк: `GET /api/chat/read-status/`; список по конкретному
   сообщению: `GET /api/chat/messages/<id>/seen/`.

## Push-уведомления о непрочитанном

После отправки сообщения бэкенд ждёт **30 секунд** и шлёт push только тем
членам группы, чей указатель чтения к этому моменту старше сообщения
(то есть кто его так и не увидел). Кто сидел в открытом чате и отметился
через `POST /read/` — push не получает. Автору push не отправляется.

- Формат: заголовок — название группы, текст — `Имя: сообщение`,
  в `data` — `{"type": "chat_message", "group_id": "..."}` для навигации по тапу.
- Уведомления одной группы коллапсируются (Android `tag`,
  iOS `apns-collapse-id`) — серия сообщений не засоряет шторку.
- Механизм best-effort: отложенная задача живёт в памяти воркера; при
  рестарте сервера в эти 30 секунд пуш не уйдёт. Если сообщение удалили
  до истечения паузы — пуш не отправляется.

### Настройка Firebase (однократно)

1. Firebase Console → **Authentication → Sign-in method** — достаточно, чтобы
   Authentication был включён (для custom tokens отдельный провайдер не нужен).
2. Firebase Console → **Firestore Database → Rules** — опубликовать правила из
   файла `firestore.rules` в корне проекта.

Custom tokens подписываются ключом сервисного аккаунта напрямую (PyJWT,
RS256), пакет `firebase-admin` не используется — он требует
`google-cloud-firestore>=2.21`, что конфликтует с pin `2.11.1` для fireo.

## GET `/api/chat/firebase-token/` — Firebase custom token

**Ответ `200 OK`:**
```json
{
    "firebase_token": "eyJhbGciOiJSUzI1NiIs...",
    "uid": "aBc123",
    "group_id": "xYz789",
    "expires_in": 3600
}
```

`uid` совпадает с ID пользователя в коллекции `users`.

**Ошибки:** `403` — нет группы, `404` — пользователь не найден,
`502` — не удалось подписать токен.

## POST `/api/chat/messages/` — Отправить сообщение

**Тело запроса:**
```json
{ "text": "Привет, группа!" }
```

`text` — обязательно, максимум 2000 символов (обрезается по пробелам по краям).

**Ответ `201 Created`:**
```json
{
    "message": "Сообщение отправлено",
    "chat_message": {
        "id": "msg123",
        "sender_id": "aBc123",
        "sender_name": "Иванов Иван",
        "sender_avatar": "https://drive.google.com/uc?id=...",
        "text": "Привет, группа!",
        "created_at": "2026-07-10 12:34:56.789012+00:00"
    }
}
```

## GET `/api/chat/messages/` — История сообщений

Сообщения своей группы, **от новых к старым**. Используется при открытии
чата (первая порция) и для подгрузки истории при прокрутке вверх.

**Query параметры:**

| Параметр | Тип    | По умолчанию | Описание                                          |
|----------|--------|--------------|---------------------------------------------------|
| `limit`  | int    | `50`         | Кол-во сообщений (максимум `100`)                 |
| `before` | string | —            | ISO-дата: вернуть сообщения строго старше неё     |

Для пагинации передайте в `before` значение `created_at` самого старого
из уже загруженных сообщений.

**Ответ `200 OK`:**
```json
{
    "total": 50,
    "has_more": true,
    "messages": [ { "id": "...", "sender_id": "...", "text": "...", "created_at": "..." } ]
}
```

## POST `/api/chat/read/` — Отметить чат прочитанным

Ставит указатель чтения текущего пользователя на текущий момент.

**Ответ `200 OK`:**
```json
{ "message": "Чат отмечен прочитанным", "last_read_at": "2026-07-10 12:40:00.000000+00:00" }
```

## GET `/api/chat/messages/<message_id>/seen/` — Кто посмотрел сообщение

Члены группы, видевшие сообщение (указатель чтения не старше `created_at`);
автор сообщения в список не входит.

**Ответ `200 OK`:**
```json
{
    "total": 1,
    "seen_by": [
        {
            "user_id": "dEf456",
            "user_name": "Петрова Анна",
            "user_avatar": "https://...",
            "last_read_at": "2026-07-10 12:40:00.000000+00:00"
        }
    ]
}
```

## DELETE `/api/chat/messages/<message_id>/` — Удалить сообщение

Автор удаляет своё сообщение; админ — любое. Админ может передать
`?group_id=<id>` для модерации чужой группы (без параметра берётся его
собственная группа). Удаление логируется в аудит.

**Ответ `200 OK`:**
```json
{ "message": "Сообщение удалено" }
```

**Ошибки:** `400` — не определить группу, `403` — чужое сообщение,
`404` — сообщение не найдено.
